# Price Tracker

Проект Price Tracker представляет собой веб-приложение, разработанное с использованием фреймворка Django, для отслеживания цен на различные товары. Он предоставляет функциональность по добавлению, редактированию, архивированию и просмотру предметов, а также отображению истории изменения цен.

## Установка и настройка

Клонируйте репозиторий:

```bash
git clone <URL репозитория>
cd price-tracker

Установите зависимости:

pip install -r requirements.txt

Примените миграции:

python manage.py migrate

Запустите сервер:

python manage.py runserver

Посетите http://127.0.0.1:8000/ в вашем веб-браузере.





Использование
Добавление предмета: Перейдите на главную страницу http://127.0.0.1:8000/. Нажмите “Добавить предмет” и введите необходимую информацию о предмете.
Редактирование и удаление предмета: На главной странице перейдите к списку предметов. Для редактирования предмета выберите “Редактировать” рядом с соответствующим предметом. Для удаления предмета выберите “Удалить” рядом с соответствующим предметом.
Обновление цен: На странице предмета можно обновить текущую цену, нажав “Обновить цену”. Это также происходит автоматически каждые 2 секунды.
История цен: На странице предмета вы можете просмотреть историю изменения цен, выбрав “История цен”.
Архивация и восстановление предметов: Предметы можно архивировать и восстанавливать. Архивированные предметы не отображаются на главной странице.
Обновление цен всех предметов: Нажмите “Обновить цены” на главной странице для массового обновления цен всех предметов.
Структура проекта
price_tracker/: Основное приложение Django.
models.py: Определение моделей данных.
views.py: Определение представлений для отображения данных.
forms.py: Определение форм для ввода данных.
urls.py: Определение маршрутов URL.
static/: Статические файлы, такие как таблицы стилей (CSS) и JavaScript.
templates/: HTML-шаблоны для отображения страниц.
main.js: JavaScript-скрипты для обновления цен и другой функциональности на стороне клиента.
price_history.js: JavaScript-скрипты для отображения истории цен с использованием библиотеки Chart.js.
Модели
Item
Модель предмета, представляющая товар, который можно купить или продать.

Поля:

name: Название товара (макс. длина 200 символов, уникальное).
link: Ссылка на товар (опционально).
image: Текстовое поле для изображения товара (опционально).
quantity: Количество товара (целое число, по умолчанию 0).
total_price: Общая стоимость товара (вещественное число, по умолчанию 0.0).
current_price: Текущая цена товара (вещественное число, по умолчанию 0.0).
spread: Разница в цене (вещественное число, по умолчанию 0.0).
last_deal_time: Время последней сделки (опционально).
target: Целевая цена товара (вещественное число, по умолчанию 0.0).
Мета:

Сортировка по убыванию времени последней сделки.
ItemAddition
Модель добавления сделки с товаром.

Поля:

item: Внешний ключ на модель Item.
transaction_type: Тип сделки (BUY - покупка, SELL - продажа).
quantity: Количество товара (целое число).
price_per_item: Цена за единицу товара (вещественное число).
commission: Комиссия в процентах (целое число, по умолчанию 13).
total: Общая стоимость сделки (вещественное число, по умолчанию 0.0).
date: Время сделки.
archived: Флаг архивации сделки (булево значение, по умолчанию False).
Мета:

Сортировка по убыванию времени сделки.
PriceHistory
Модель истории цен для каждого товара.

Поля:

item: Внешний ключ на модель Item.
price: Цена товара (вещественное число).
timestamp: Временная метка создания записи.
Мета:

Сортировка по времени.
Формы
ItemAdditionForm
Форма для добавления сделки.

Поля:

item: Выбор товара.
transaction_type: Выбор типа сделки.
quantity: Количество товара.
price_per_item: Цена за единицу товара.
commission: Комиссия в процентах.
date: Дата и время сделки (опционально).
seconds: Скрытое поле для хранения секунд времени сделки (опционально).
Методы:

clean: Очистка и проверка данных формы.
clean_date: Очистка и проверка поля ‘date’.
clean_quantity: Очистка и проверка поля ‘quantity’.
clean_price_per_item: Очистка и проверка поля ‘price_per_item’.
ItemForm
Форма создания нового товара.

Поля:

name: Название товара.
link: Ссылка на товар (опционально).
image: Текстовое поле для изображения товара (опционально).
Технологический стек

Chart.js: 2.x
jQuery: 3.x
